2. Step-by-Step Teaching Flow
Step 1: Naive Approach (Kill it)
Idea:
Store all web pages in list
Search each page for keyword
Complexity:
O(N × content size)

👉 Show failure:

Billions of pages → impossible
Step 2: First Optimization → Indexing
Introduce:

“Don’t search documents. Search an index.”

Concept:
Word → list of documents

👉 This leads to:

HashMap (Inverted Index)
Teach DSA:
Key = word
Value = list of URLs

Example:

"rust" → [doc1, doc20, doc105]
Step 3: Fast Lookup → Hashing

Now ask:

“How do we find ‘rust’ instantly?”

👉 Introduce:

Hash function
O(1) lookup
Step 4: Prefix Search → Trie

Now extend:

“When you type ‘ru’, suggestions appear…”

👉 Introduce:

Trie (Prefix Tree)
Key Learning:
Each character → node
Fast prefix search
Step 5: Ranking Results → Heap

Ask:

“Why best results appear first?”

👉 Introduce:

Ranking (PageRank concept simplified)
Top-K problem

👉 Use:

Heap / Priority Queue
Step 6: Relationships → Graph

Now go deeper:

“How does Google know which page is important?”

👉 Introduce:

Web as graph
Links = edges

👉 Concept:

Graph + ranking algorithm
3. DSA Covered (Clean Mapping)
Problem	DSA
Fast word lookup	HashMap
Autocomplete	Trie
Top results	Heap
Web relationships	Graph
Efficient storage	Indexing
4. System Design Bridge (Critical)

Now transition:

Ask:

“What happens when data = billions of pages?”

Introduce Concepts:
1. Distributed Index
Split data across servers
2. Caching
Popular searches cached
3. Load Balancing
Requests distributed
4. Parallel Processing
Search across shards
Final Architecture (Simple Flow)

User → Load Balancer → Search Servers → Index → Rank → Results
