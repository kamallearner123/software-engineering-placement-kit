# Module 3: System Design

**Duration:** 6+ hours  
**Focus:** System design fundamentals, Scalability, HLD, and LLD.

## 📋 Module Overview

This module introduces you to system design concepts essential for building scalable software systems. Learn about load balancing, caching, CAP theorem, and deep-dive into High-Level Design (HLD) case studies.

## 🎯 Learning Objectives

- Understand fundamental system design concepts.
- Learn scalability patterns and trade-offs.
- Master High-Level Design (HLD) for interview problems.
- Understand Low-Level Design (LLD) and Design Patterns.
- Understand the software development lifecycle (SDLC).

## 📚 Topics Covered

### 1. Introduction to System Design (1.5 hours)
- **System Design Terminology**
  - Scalability (Horizontal vs Vertical)
  - Availability (Redundancy, Failover)
  - Load Balancing (Nginx, HAProxy)
  - Caching Strategies (Redis, Memcached)
- **CAP Theorem**
  - Consistency, Availability, Partition Tolerance
  - Trade-offs in Distributed Systems

### 2. High-Level Design (HLD) Case Studies (2 hours)
- **Common Interview Designs**
  - URL Shortener (TinyURL)
  - Chat System (WhatsApp)
  - Rate Limiter
  - News Feed (Twitter/Facebook)
- **[View HLD Case Studies](./05_HLD_Case_Studies/README.md)**

### 3. Low-Level Design (LLD) & Design Patterns (2 hours)
- **SOLID Principles**
- **Creational Patterns**: Singleton, Factory.
- **Structural Patterns**: Adapter, Proxy.
- **Behavioral Patterns**: Observer, Strategy.

### 4. SDLC - Software Development Life Cycle (30 minutes)
- Planning, Design, Development, Testing, Deployment, Maintenance.
- Common SDLC Models (Waterfall, Agile).

## 🔨 Practical Exercises

- Design a scalable web application from scratch.
- Implement a Singleton pattern in C++ or Java.
- Draw an architecture diagram for a real-time chat application.

## 🌟 Key Interview Topics

- Horizontal vs Vertical scaling.
- CAP theorem and trade-offs.
- SQL vs NoSQL decision making.
- Microservices vs Monolith.

## 💡 Pro Tips

- **Think in Trade-offs:** Every design decision has pros and cons.
- **Draw Diagrams:** System design interviews require visual explanations.
- **Estimate Scale:** Learn how to calculate QPS (Queries Per Second) and storage.
